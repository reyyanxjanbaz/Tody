# Tody Project Context

## 1. Project Overview & Philosophy
**Tody** is a "human-centric" to-do list application that focuses on respecting human psychology. It is intended to reduce friction and cognitive load, with a premium, gesture-driven interface.

### Key Philosophies & Features:
- **Magnetic Deadline Snapping**: Snapping time pickers to logical times (e.g., 9:00 AM) with haptic feedback.
- **The Today Line**: A UI element separating "Now" from "Later".
- **Pull-to-Focus**: Pulling down the list enters a distraction-free mode (showing top 3 tasks).
- **Swipe Action Memory**: The app learns and reorders swipe actions based on user habits.
- **Contextual Empty States**: Suggestions on what to do rather than a blank screen.
- **Process Inbox (GTD)**: A mode for rapid triage of raw thoughts into structured tasks.
- **Reality Score (Experimental)**: A metric measuring planning accuracy vs. completion history.
- **Smart Keyboard Toolbar**: Context-aware options when typing.

## 2. Tech Stack Overview
- **Mobile Client**: React Native 0.84 (New Architecture), React 19, TypeScript
  - **Libraries**: `react-native-reanimated`, `@shopify/flash-list`, `react-native-gesture-handler`, `react-native-haptic-feedback`
  - **Navigation**: React Navigation v7
  - **State/Storage**: React Context API + `@react-native-async-storage/async-storage` for offline persistence.
- **Backend Server**: Python 3.10+ FastAPI (located in `/server`)
  - **Endpoints**: Used for specialized business logic, analytics, sync API extensions (`/tasks`, `/categories`, `/inbox`, `/patterns`, `/profile`)
- **Database**: Supabase (PostgreSQL)
  - **Features Leveraged**: Row Level Security (RLS) policies, Storage (for avatars), triggers for caching logic (e.g., task completion tracking).

## 3. Data Flow & Sync Architecture
The app features an offline-first architecture with background synchronization:
1. **Local State**: Context (`TaskContext.tsx`, `AuthContext.tsx`, `InboxContext.tsx`) manages all local state. Data is written to `AsyncStorage` via debounced saves (`persistTimeout`).
2. **Supabase Sync Strategy**:
   - Initial sync on login: Existing user DB state is fetched and merged with local state (DB wins state conflicts based on `updatedAt`).
   - Mutations: Write local first, update UI instantly (LayoutAnimation), then fire async requests to the API endpoints. If backend is unreachable, it falls back to direct `upsert` via Supabase client library (`src/lib/supabase.ts` / `supabaseSync.ts`).
   - `buildCategoryMap`: Ensures client string IDs for categories (`work`, `health`) map to UUIDs in the database correctly.

## 4. Key Data Models (Supabase Schema)
- **Profiles (`auth.users` extension)**: User preferences, `dark_mode`, date/time formats, `display_name`, and `swipe_stats` (JSONB) for swipe action learning.
- **Categories (Customized Task Groups)**:
  - Default: Overview, Work, Personal, Health. Custom categories can be created per user.
- **Tasks**:
  - `priority`: 'high', 'medium', 'low', 'none'
  - `energy_level`: 'high', 'medium', 'low'
  - `defer_count`: Tracks how many times a task is pushed to tomorrow.
  - `estimated_minutes` vs `actual_minutes`: For checking planning accuracy.
  - `is_completed`, `completed_at`, `is_archived`, `archived_at`
  - Support for recurring tasks (`is_recurring`, `recurring_frequency`).
  - **Dependency Chain**: Support up to 3 depth levels of nested tasks (`parent_id`, `child_ids`).
- **Inbox Tasks**: For rapid GTD capture (`raw_text`, `captured_at`).
- **Task Patterns**: Tracks time estimation learning using `keywords`, `average_actual_minutes`, and `accuracy_score`.

## 5. Mobile Application Structure (`/src`)
- **`/navigation`**: Standard setup with `RootNavigator` managing authenticated paths (Screens like Home, Archive, TaskDetail, ProcessInbox, Profile, Settings, RealityScore) versus unauthenticated paths (Login, Register).
- **`/context`**: 
  - `AuthContext.tsx`: Manages authentication flows (Supabase auth wrapper).
  - `TaskContext.tsx`: Core brain of the application for Tasks. Extensively manages optimistic updates and Supabase sync logic.
  - `InboxContext.tsx`, `ThemeContext.tsx`, `UndoToast.tsx` (`Provider`)
- **`/screens`**:
  - `HomeScreen`: Shows tasks.
  - `TaskDetailScreen`: Dedicated edit page.
  - `RealityScoreScreen`: Metrics overview for the new Reality Score feature.
  - `ProcessInboxScreen`: Quick triage functionality.
- **`/components`**: Reusable parts. Notables: `DeadlineSnapper.tsx`, `FocusOrb.tsx`, `TaskPreviewOverlay.tsx`, `TodayLine.tsx`, `EnergyFilter.tsx`.

## 6. Server / Backend (`/server`)
- **`main.py`**: FastAPI entrypoint initializing middleware and global handlers.
- **`db.py`**: Supabase service role / regular role client instances.
- **`auth.py`**: Validates Authorization bearer tokens generated by Supabase.
- **Routers**:
  - `tasks.py`: Full CRUD along with specialty endpoints like `/sync`, `/batch`, `/{id}/complete`, `/{id}/defer` which offload backend processing and reduce client bandwidth usage.
  - `profile.py`: For metrics calculation.
  - `patterns.py`: Task learning APIs.

## 7. Known System Patterns / Features to Observe
- **Auto-completion logic**: In `tasks.py` and Postgres Triggers, marking a task `is_completed = true` conditionally sets `completed_at` to `now()` or figures out the `actual_minutes` automatically.
- **Layout Animations**: Native iOS/Android `LayoutAnimation` framework is actively managed alongside `react-native-reanimated` transitions for a calm vibe.
